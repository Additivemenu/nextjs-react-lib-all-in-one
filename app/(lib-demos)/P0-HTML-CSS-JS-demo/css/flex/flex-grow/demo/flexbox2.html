üéì Key Takeaway

The bug only occurs when:

  1. You have a flex item (with flex: 1)
  2. That contains nested content (like a <div> or <Truncate> component)
  3. The nested content needs to truncate with ellipsis

Why min-width: 0 fixes it:

  • By default, flex items have min-width: auto
  • This means "don't shrink smaller than my content"
  • For nested truncatable content, the flex item sees the full text width as its minimum
  • min-width: 0 overrides this and says "you CAN shrink below your content size"
  • This allows the nested text-overflow: ellipsis to actually work
